HOW THE DATASET FLOWS THROUGH THE SYSTEM
────────────────────────────────────────────────────────────────────────────

USER INPUT (Theme + Text/File)
         ↓
    [cominote_engine.generate()]
         ↓
    [Collect & Parse Input]
         ↓
    [Extract Concepts & Sentences]
         ↓
    [Build Story Scenes]
         ↓
    [_build_visual_plan()]  ← ⭐ DATASET SELECTION HAPPENS HERE
         ↓
    1. Load theme profile from dataset
    2. Select characters from _dataset_characters filtered by theme
    3. Select backgrounds from _dataset_backgrounds filtered by subject
    4. Pick emotions from _dataset_emotions (sequence based on panels)
    5. Pick poses from _dataset_poses (matching emotion + theme)
    6. Pick speech bubbles from _dataset_bubbles
    7. Pick sound effects from _dataset_sfx
         ↓
    [Render Comic Image] ← Uses all selected dataset elements
         ↓
    [Save PNG + Metadata]
         ↓
    OUTPUT: Beautiful Comic with Dataset Characters & Colors

DATASET FEATURES EXPLAINED
────────────────────────────────────────────────────────────────────────────

1. CHARACTERS.JSON (5 anime + 5 pixar + 2 superhero + 3+ cartoon)
   Each character has:
   - name: The character's name
   - category: "student", "teacher", "mascot", "subject_expert"
   - sub_type: "anime_learner", "pixar_student", "superhero", "cute_cartoon"
   - theme_fit: ["anime"], ["pixar"], ["superhero"], etc.
   - subject_fit: ["science"], ["history"], ["literature"], ["all"], etc.
   - image_prompt: How to generate this character visually
   - palette: {hair, skin, costume_primary, costume_accent, eye}
   - costume: {top, bottom, accessory, shoes}

   🎯 USAGE: ComicText → Theme Selected → Characters Filtered → 2-4 Picked

2. BACKGROUNDS.JSON (15+ themed locations)
   Each background has:
   - name: Location name
   - category: "school", "science", "nature", "fantasy", "historical"
   - theme_fit: Which themes use this background
   - subject_fit: Which subjects need this background
   - prompt_fragment: Visual description
   - palette: {primary, secondary, accent, floor}

   🎯 USAGE: Subject → Theme → 3-6 Backgrounds Selected & Rendered

3. EMOTIONS.JSON (6 core emotions)
   Each emotion has:
   - name: "happy", "thinking", "excited", "surprised", "shocked", "proud"
   - sound_effect: "YES!", "AHA!", "WOW!", etc.
   - prompt_fragment: Visual description

   🎯 USAGE: Panel Position → Emotion Assigned → Affects facial expression

4. POSES.JSON (8+ character poses)
   Each pose has:
   - name: "standing", "explaining", "thinking", "celebrating", etc.
   - prompt_fragment: Visual description
   - good_for_emotions: Which emotions work with this pose
   - theme_fit: Which themes can use this pose

   🎯 USAGE: Emotion → Pose Selected → Character drawn in that pose

5. THEME_ASSET_PROFILES.JSON (6 themes)
   Each profile defines:
   - slug: "anime", "pixar", "superhero", "cartoon_kids", "manga", "fantasy"
   - label: Human-readable name
   - dataset_style: Visual rendering style
   - palette_style: Color scheme
   - layout_style: Panel arrangement
   - preferred_bubble: Speech bubble type
   - fonts: UI fonts (title, body, effect)
   - character_summary: Character description
   - background_summary: Background description

   🎯 USAGE: User Selects Theme → All theme parameters applied

6. PANELS_TEXT_STYLES.JSON (Layouts, Bubbles, Effects, Styles)
   Defines:
   - panel_layouts: "balanced_grid", "cinematic_grid", "action_splash", "manga_cascade"
   - speech_bubbles: "normal", "pixar_round", "shouting", "thought_bubble", "manga_box"
   - sound_effects: "POW!", "WOW!", "AHA!", "BOOM!", "YES!", "ZAP!"
   - styles: Named visual styles

   🎯 USAGE: Theme Determines → Layout, Bubble Type, SFX Colors Selected

TROUBLESHOOTING GUIDE
────────────────────────────────────────────────────────────────────────────

Problem: Dataset not loading
Solution:
  1. Check /dataset/ folder exists and has all 6 JSON files
  2. Run: ls -la /Users/kottakkisaisrija/Desktop/mini\ project/dataset/
  3. Verify JSON syntax: python -m json.tool dataset/characters.json
  4. Restart app: python app.py

Problem: Characters not showing in comics
Solution:
  1. Check _dataset_characters is populated:
     python -c "from cominote_engine import CominoteEngine; from pathlib import Path; e = CominoteEngine(Path('.')); print(len(e._dataset_characters))"
  2. Verify character IDs in JSON files
  3. Check theme profile exists: _theme_profiles_by_slug

Problem: Colors look muted, not vibrant
Solution:
  1. Check character palettes in characters.json
  2. Verify hex colors are valid (#RRGGBB format)
  3. Check theme palette_style is correct
  4. Regenerate comic with different theme

Problem: UI looks broken after patches
Solution:
  1. Clear browser cache: Cmd+Shift+Delete
  2. Verify file edits were saved correctly
  3. Check for missing closing tags in HTML
  4. View page source to see actual HTML
  5. Check browser console for JavaScript errors (F12)

Problem: Theme dropdown shows "Choose a theme" but stays empty
Solution:
  1. Check /api/themes endpoint: curl http://localhost:5001/api/themes
  2. Verify theme JSON files load correctly
  3. Check app.js loadThemeOptions() function
  4. Check browser console for fetch errors
